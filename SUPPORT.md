# Support

We are glad you are using **Laravel Query Engine** (`victormgomes/laravel-query-engine`)!

If you need help or want to discuss a feature, please keep the following
guidelines in mind to help us maintain the repository effectively.

## 📦 Release Policy

To ensure the package evolves rapidly and remains maintainable, we strictly follow a **Latest Major Version Only** support policy:

- **Only the latest major version receives updates.**
- All new features, bug fixes, and security patches are released exclusively on the `main` branch under the latest major version.
- Older major versions are not maintained. If you encounter a bug on an older version, please upgrade to the latest version to receive the fix.

## 🐛 Bug Reports & Feature Requests

If you have found a verifiable bug or have a well-thought-out feature request,
please [open an issue](../../issues/new/choose) on our GitHub repository.

Before opening a new issue, please **search the existing issues** to see if your
problem has already been reported or solved.

## 💬 General Questions & Discussions

**Please do not use the issue tracker for personal support requests, debugging
your own code, or general questions.**

If you are having trouble getting the package to work, have a specific question
about an architecture integration, or simply want to share an idea, please use
the [GitHub Discussions](../../discussions) tab. This helps us keep the issue
tracker clean and focused strictly on actionable code changes.

## 🚨 Security Vulnerabilities

If you discover a security vulnerability within this package, please review our
[SECURITY.md](SECURITY.md) policy and email the author directly at
**<info@victormgomes.net>**. All security vulnerabilities will be promptly
addressed.
